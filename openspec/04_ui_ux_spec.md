# SonicVision AI UI/UX 设计规范

## 1. 交互规范
### 1.1 状态转换
- **迷你工具栏 (Mini Bar):**
  - 常态: 悬浮于底部，100% 不透明。
  - 闲置状态 (3s 无操作): 降至 15% 不透明度 (`opacity-[0.15]`)。
- **设置面板 (Expanded Control Panel):**
  - 展开后锁定 90% 不透明度 (`bg-[#060608]/90`)，不受闲置状态影响。

### 1.2 快捷键响应
- `Space`: 切换音频采集。
- `R`: 随机化视觉模式与配色。
- `F`: 切换全屏。
- `L`: 切换歌词显示。
- `H`: 隐藏/显示 UI。
- `Arrow Keys`: 循环切换模式与配色主题。

## 2. 视觉表现 (Visual Styles)
### 2.1 配色方案 (Color Themes)
- 预设 **28 套** 艺术配色。
- 采用数组形式存储 (`string[]`)，渲染器自动进行渐变插值或循环索引。

### 2.2 歌词动效 (Typography)
- **Karaoke Style:** 
  - 字体: `Inter`, Weight `900`。
  - 动效: 基于 `bassNormalized` 的实时缩放 (`scale`)。
- **Minimal Style:** 
  - 字体: `monospace`, Weight `300`。
  - 动效: 字符间距扩充，基于振幅的透明度呼吸效果。

## 3. 国际化 (i18n)
- **支持语言:** 英语、中文、日语（通过 `translations.ts` 管理）。
- **区域定位 (Region):** 影响 AI 识别的搜索偏移，默认为浏览器语言所对应地区。

## 4. 无障碍与性能
- **滑块控制:** 优化 `input[type=range]` 的 Webkit 样式，增加点击区域及视觉反馈。
- **性能提示:** 在设置面板中提示“发光 (Glow)”效果对性能的影响。
